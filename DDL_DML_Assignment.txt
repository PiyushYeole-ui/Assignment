üéØDDL (Data Definition Language)

üìåDDL ‚Äì Definition
DDL (Data Definition Language) is used to create, modify, and delete database structures such as tables, schemas, indexes, and constraints.
DDL commands change the structure of the database ‚Äî not the data inside it.

Main Purpose :-

       To create database objects

       To change structure of existing objects

       To remove/rename objects

       Works on schema, not on data


Types of DDL Commands + Definition + SQL Code

1. CREATE
Definition: Creates a new database object (table, view, schema, etc.)

  ‚û°Ô∏èSQL Example (RUNNABLE CODE):

  CREATE TABLE Students (
      RollNo INT PRIMARY KEY,
      Name VARCHAR(50),
      Age INT,
      City VARCHAR(50)
  );


2. ALTER

Definition: Modifies an existing database object.

  ‚û°Ô∏èSQL Example (RUNNABLE CODE):

  ALTER TABLE Students ADD Email VARCHAR(100);
  ALTER TABLE Students MODIFY Age INT NOT NULL;
  ALTER TABLE Students DROP COLUMN City;

3. DROP

Definition: Completely deletes a database object.

  ‚û°Ô∏èSQL Example (RUNNABLE CODE):

      DROP TABLE Students;

4. TRUNCATE

Definition: Deletes all rows of a table but keeps structure intact

  ‚û°Ô∏èSQL Example (RUNNABLE CODE):

   TRUNCATE TABLE Students;


5. RENAME

Definition: Changes the name of a database object.

  ‚û°Ô∏èSQL Example (RUNNABLE CODE):
  
	 RENAME TABLE Students TO College_Students;
	 
6. COMMENT (Optional but valid DDL)

Definition: Adds comments/documentation to database objects.

  Ô∏è ‚û°Ô∏èSQL Example (RUNNABLE CODE):
  
           COMMENT ON TABLE College_Students IS 'Contains student academic records';


Key Features :-
          Mostly auto-commit (once executed, rollback isn‚Äôt possible)
          Affects structure, not individual rows
          Fast execution since no row-level logs like DML

------------------------------------------

| -------- | --------------------------- |

| DROP     | Deletes objects permanently |

	 üìå DML ‚Äì Data Manipulation Language
	 Definition:-  DML is the part of SQL used to insert, retrieve, update and delete data stored inside database tables. 
	 It doesn‚Äôt change table structure ‚Äî only the records
	 
	 DML Commands (Types + Definition + SQL Code)
	 1) SELECT
        Definition: Used to retrieve/read data from a table
		‚û°Ô∏èSQL Example (RUNNABLE CODE):
		  
		  SELECT * FROM employees;
          SELECT name, salary FROM employees WHERE salary > 30000;
       
	  2) INSERT
        Definition: Used to add new rows/data into a table
		‚û°Ô∏èSQL Example (RUNNABLE CODE):
		INSERT INTO employees (id, name, salary)
        VALUES (101, 'Riya', 45000),
               (102, 'Aman', 52000);
 
      3) UPDATE
         Definition: Used to modify existing data inside a table.
		 ‚û°Ô∏èSQL Example (RUNNABLE CODE):
		 
		 UPDATE employees
         SET salary = salary + 5000
         WHERE id = 102;
		 
	 4) DELETE
        Definition: Used to remove records permanently from a table.
		 ‚û°Ô∏èSQL Example (RUNNABLE CODE):
		 DELETE FROM employees WHERE id = 101;


Summary Table:- 

-------------------------------------------------------------------------------------------------------
| Command    | Definition / Purpose          | Basic SQL Syntax Example                            |
| ---------- | ----------------------------- | --------------------------------------------------- |
| **SELECT** | Retrieve/read data from table | `SELECT * FROM students;`                           |
| **INSERT** | Add new records into table    | `INSERT INTO students (id,name) VALUES (1,'Amit');` |
| **UPDATE** | Modify existing data          | `UPDATE students SET name='Riya' WHERE id=1;`       |
| **DELETE** | Remove records permanently    | `DELETE FROM students WHERE id=1;`                  |
--------------------------------------------------------------------------------------------------------
